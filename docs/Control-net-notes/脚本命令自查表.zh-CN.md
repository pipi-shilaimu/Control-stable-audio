# StableAudio ControlNet 脚本命令自查表

本文档整理本项目里常用脚本命令的运行顺序、自查点和使用示例。默认环境为 Windows PowerShell，工作目录为仓库根目录：

如果在 Linux/bash 或 Docker 容器里运行，请把路径分隔符改成 `/`，续行符改成 `\`。不要直接复制 PowerShell 示例里的反引号 `` ` ``，它在 bash 中不是续行符。

```powershell
cd C:\PROJECT\StableAudio
```

命令示例统一使用本项目虚拟环境：

```powershell
.\.venv\Scripts\python.exe <script.py> <args>
```

## 0. 总览：先跑哪一个

| 阶段 | 推荐脚本 | 目的 | 通过后说明 |
| --- | --- | --- | --- |
| 环境自查 | `stable_audio_control/scripts/main.py` | 检查 PyTorch、CUDA、GPU、显存和矩阵乘法 | 当前 Python 环境能用 GPU |
| 导入自查 | `python -c "import ..."` | 检查关键包能否导入 | `.venv` 和本地包路径基本正确 |
| 数据导出 | `stable_audio_control/data/dataset_load.py` | 把 Hugging Face 数据集转成 `audio_dir + custom_metadata_module` | 训练脚本能读取 wav 和 prompt |
| 最小控制注入 | `stable_audio_control/scripts/smoke_control_injection.py` | 不加载真实 StableAudio，先验证 ControlNet transformer 逻辑 | zero-linear 注入路径结构正确 |
| 真实 wrapper 冒烟 | `stable_audio_control/scripts/smoke_control_dit_wrapper.py` | 加载真实 StableAudio Open 并验证 `melody_control` 被消费 | ControlNet-DiT wrapper 链路可用 |
| 单步训练冒烟 | `stable_audio_control/scripts/train_control_smoke.py` | 跑一轮 `training_step + backward + optimizer.step` | loss 有限，冻结策略正常 |
| 正式短训 | `stable_audio_control/scripts/train_controlnet_dit.py` | 训练 ControlNet 分支 | 能产出 Lightning checkpoint |
| 最小生成对比 | `stable_audio_control/scripts/compare_controlnet_generation.py` | 同 prompt 下生成 base/control 两份音频 | 控制路径对生成结果有影响 |
| 旋律相似度比较 | `stable_audio_control/scripts/compare_melody_similarity.py` | 比较参考音频和生成音频的 melody feature 接近程度 | 判断 control 是否真的更贴近参考旋律 |
| 批量随机对比 | `stable_audio_control/scripts/compare_controlnet_generation_random.py` | 随机抽 10 个参考音频和 10 个 seed，逐条生成并打分 | 快速看一批样本的平均旋律贴合度 |
| 旋律可视化 | `scripts/fft_cqt_visual_compare.py` | 生成 FFT/CQT 对比视频或 GIF | 观察旋律/频谱特征是否合理 |
| 单元测试 | `python -m unittest discover -s tests` | 回归测试 | 代码层面的关键行为未破坏 |

## 1. 环境与导入自查

### 1.1 GPU 环境自查

脚本：

```text
stable_audio_control/scripts/main.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\main.py
```

看什么：

- `CUDA 是否可用: True`
- 能打印 GPU 名称、显存、计算能力；
- `GPU计算测试通过`；
- 若没有 CUDA，训练和生成仍可在 CPU 上调试，但 StableAudio Open 会非常慢。

常见问题：

- `CUDA 不可用`：先确认当前 `.venv` 装的是 CUDA 版 PyTorch，并检查 NVIDIA 驱动。
- GPU 能看到但训练慢：后续训练可尝试 `--precision 16-mixed`，但排查错误时建议先用 `--precision 32-true`。

### 1.2 关键包导入自查

命令：

```powershell
.\.venv\Scripts\python.exe -c "import stable_audio_tools; import stable_audio_control.models; import stable_audio_control.melody.extractors; print('IMPORT_OK')"
```

看什么：

- 输出 `IMPORT_OK`。

常见问题：

- 找不到 `stable_audio_control`：确认从仓库根目录运行，或确认脚本里已有 `sys.path.insert(...)`。
- 找不到 `stable_audio_tools`：确认 `.venv` 已安装依赖。
- `No module named 'flash_attn'`：当前项目里通常是可忽略警告，脚本会禁用 Flash Attention。

### 1.3 Hugging Face / T5 缓存自查

`stable_audio_tools.get_pretrained_model("stabilityai/stable-audio-open-1.0")` 会先下载 StableAudio Open 的 `model_config.json` 和权重，然后在构建 text conditioner 时加载 T5 tokenizer/encoder。若容器没有联网，或 Hugging Face 缓存不完整，生成脚本可能在 `AutoTokenizer.from_pretrained(...)` 处失败。

Linux / Docker bash 示例：

```bash
export HF_HOME=/outputs/hf_cache
python3 -c "from transformers import AutoTokenizer, T5EncoderModel; AutoTokenizer.from_pretrained('t5-base'); T5EncoderModel.from_pretrained('t5-base'); from stable_audio_tools import get_pretrained_model; get_pretrained_model('stabilityai/stable-audio-open-1.0'); print('HF_CACHE_OK')"
```

看什么：

- 输出 `HF_CACHE_OK`；
- 如果失败，先解决 Hugging Face 网络、`HF_TOKEN`、缓存目录权限或离线缓存同步问题；
- 如果容器离线，先在有网络的环境填好 `HF_HOME` 对应缓存，再把缓存目录挂载进容器。

## 2. 数据集导出与数据配置

### 2.1 Song Describer 导出为 `audio_dir`

脚本：

```text
stable_audio_control/data/dataset_load.py
```

用途：

- 从 Hugging Face 数据集导出 wav；
- 生成 `manifests/<split>.json`；
- 生成 `dataset_config_<split>.json`；
- 配好 `custom_metadata_module`，让训练时能从音频文件取回 prompt。

小规模试导出：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\data\dataset_load.py `
  --output-dir stable_audio_control\data\song_describer `
  --splits train `
  --valid-only `
  --max-items 20
```

较完整导出：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\data\dataset_load.py `
  --output-dir stable_audio_control\data\song_describer `
  --splits train `
  --valid-only
```

输出应包含：

```text
train: exported <N> files
  audio_dir: ...
  manifest: ...
  dataset_config: ...
```

重点检查：

- `stable_audio_control/data/song_describer/train/*.wav` 是否存在；
- `stable_audio_control/data/song_describer/manifests/train.json` 是否存在；
- `stable_audio_control/data/song_describer/dataset_config_train.json` 里的 `path` 指向真实音频目录；
- `custom_metadata_module` 指向 `stable_audio_control/data/song_describer_metadata.py`。

常见问题：

- `Audio mapping must contain an 'array' key`：当前脚本已兼容 `array`、`bytes`、`path` 三种形式；若再次出现，先看数据集字段结构是否变了。
- Hugging Face 下载慢：可配置 `HF_TOKEN` 或先让数据集缓存完整。
- 只导出了部分文件：检查是否传了 `--max-items`。

### 2.2 `custom_metadata_module` 自查

脚本：

```text
stable_audio_control/data/song_describer_metadata.py
```

它不是直接运行的脚本，而是被 `stable_audio_tools` 的 `audio_dir` 数据集加载器调用。

它做的事：

- 根据当前音频路径寻找同级 `manifests/<split>.json`；
- 用 `relpath` 或文件名匹配 manifest；
- 返回至少包含 `prompt` 的 metadata；
- 把 Song Describer 额外字段写成 `song_describer_<key>`。

检查 `dataset_config_train.json` 示例：

```json
{
  "dataset_type": "audio_dir",
  "datasets": [
    {
      "id": "song_describer_train",
      "path": "C:\\PROJECT\\StableAudio\\stable_audio_control\\data\\song_describer\\train",
      "custom_metadata_module": "C:\\PROJECT\\StableAudio\\stable_audio_control\\data\\song_describer_metadata.py"
    }
  ],
  "random_crop": true
}
```

Windows 多进程注意：

- `train_controlnet_dit.py` 已处理 `custom_metadata_module` 的 pickling 问题；
- `custom_metadata_module` 最好放在本仓库包路径内，例如 `stable_audio_control/data/song_describer_metadata.py`；
- 若 DataLoader worker 报 pickling 错，优先确认 `custom_metadata_module` 是否能作为包模块导入。

## 3. ControlNet 冒烟测试

### 3.1 最小 transformer 注入测试

脚本：

```text
stable_audio_control/scripts/smoke_control_injection.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\smoke_control_injection.py
```

用途：

- 不加载 StableAudio Open；
- 用小型 `ContinuousTransformer` 包上 `ControlNetContinuousTransformer`；
- 比较 `control_scale=0.0` 和 `control_scale=0.5`；
- 验证 zero-linear 初始为 0 时输出不变，轻微扰动 adapter 后输出变化。

通过标志：

- 第一段 `diff norm` 接近 `0`；
- 扰动 adapter 后 `diff norm > 0`。

适用场景：

- 刚改过 `control_transformer.py`；
- 想快速验证控制注入路径，没有必要加载大模型。

### 3.2 真实 StableAudio transformer 替换测试

脚本：

```text
stable_audio_control/scripts/smoke_control_injection_stableaudio_open-1.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\smoke_control_injection_stableaudio_open-1.py
```

用途：

- 加载 `stabilityai/stable-audio-open-1.0`；
- 把内部 `ContinuousTransformer` 替换为 `ControlNetContinuousTransformer`；
- 直接跑一次前向。

看什么：

- 能打印 `out0`、`out1` 形状和 norm；
- 默认 zero-linear 未扰动时，`diff norm` 可以是 `0`，这是预期行为。

常见问题：

- 首次运行需要 Hugging Face 模型缓存；
- T5 tokenizer/model 未缓存时可能卡在下载；
- 显存不足时优先用下面的 wrapper smoke 或缩短后续生成长度。

### 3.3 ControlNet-DiT wrapper 测试

脚本：

```text
stable_audio_control/scripts/smoke_control_dit_wrapper.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\smoke_control_dit_wrapper.py
```

用途：

- 验证 `build_control_wrapper(...)` 能包装真实 StableAudio Open；
- 验证 `cond["melody_control"]` 会被 `ControlConditionedDiffusionWrapper` 消费；

### 3.4 单步训练 smoke

脚本：

```text
stable_audio_control/scripts/train_control_smoke.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\train_control_smoke.py
```

用途：

- 构造最小随机音频 batch；
- 在线提取 CQT top-k；
- 执行 `training_step`、`backward`、`optimizer.step`；
- 检查 loss、梯度、冻结策略和参数更新。

通过标志：

```text
loss_is_finite: True
frozen_with_grad_count: 0
trainable_probe ... delta: <大于 0>
frozen_probe ... delta: 0.0
Smoke PASS: one training step completed with finite loss and expected freeze behavior.
```

失败先查：

- `Attempted to use an uninitialized parameter`：检查 lazy 参数初始化；

- 冻结参数出现梯度：检查冻结策略是否误放开 base transformer。

## 4. 正式训练入口

### 4.1 默认 CQT 训练

脚本：

```text
stable_audio_control/scripts/train_controlnet_dit.py
```

最小短训命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\train_controlnet_dit.py `
  --dataset-config .\stable_audio_control\data\song_describer\dataset_config_train.json `
  --max-steps 50 `
  --batch-size 1 `
  --num-workers 1 `
  --precision 32-true
```

它训练什么：

- 训练 `control_layers`；
- 训练 `zero_linears`；
- CQT 路径下训练 `melody_encoder`；
- fallback/dense 路径相关的 `control_projector` 会被初始化并纳入冻结策略边界；
- 默认冻结 StableAudio Open 主干。

启动后重点看：

```text
model_name=stabilityai/stable-audio-open-1.0
melody_feature=cqt, control_channels=8
sample_rate=44100, sample_size=...
trainable_modules=<N> tensors
trainable_name_samples=[...]
```

训练中重点看：

- `train/loss` 是有限值；
- step 能持续前进；
- `max_steps=50` 到达后能正常保存 checkpoint。

checkpoint 常见位置：

```text
outputs/train_controlnet_dit/lightning_logs/version_<N>/checkpoints/epoch=0-step=50.ckpt
```

常见问题：

- `LRScheduler.__init__()` 参数不兼容：脚本已 patch `stable_audio_tools` 的 `InverseLR`；
- Windows DataLoader pickling 错：脚本已尝试把 `custom_metadata_module` 替换成包级导入；确认 metadata 文件在包路径内；
- `padding_mask` 维度错误：脚本已把裸 tensor 包成上游期望的 list；
- `torchcodec/libtorchcodec` 加载失败：训练脚本已安装 torchaudio load fallback；若仍失败，检查音频文件是否能被 `soundfile` 读取。

### 4.2 Chromagram 训练

`chromagram` 是和 CQT 并存的可切换旋律后端，默认仍是 CQT。Chromagram 当前使用 12 维 pitch-class dense 特征。

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\train_controlnet_dit.py `
  --dataset-config .\stable_audio_control\data\song_describer\dataset_config_train.json `
  --max-steps 50 `
  --batch-size 1 `
  --num-workers 1 `
  --precision 32-true `
  --melody-feature chromagram `
  --chroma-bins 12 `
  --chroma-n-fft 2048 `
  --default-root-dir outputs\train_controlnet_dit_chromagram
```

启动后重点看：

```text
melody_feature=chromagram, control_channels=12
```

注意：

- chromagram 输出是 `[B, 12, F]` 浮点 dense 控制；
- CQT 输出默认是 `[B, 8, F]` 离散 top-k 控制；
- 对比生成时必须使用和 checkpoint 一致的 `--melody-feature` 与通道参数；
- 不建议拿 CQT 训练出的 ckpt 直接用 chromagram 对比，反之亦然。

### 4.3 续训

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\train_controlnet_dit.py `
  --dataset-config .\stable_audio_control\data\song_describer\dataset_config_train.json `
  --ckpt-path outputs\train_controlnet_dit\lightning_logs\version_5\checkpoints\epoch=0-step=50.ckpt `
  --max-steps 200 `
  --batch-size 1 `
  --num-workers 1 `
  --precision 32-true
```

自查点：

- `--ckpt-path` 指向真实存在的 `.ckpt`；
- 续训的 `--melody-feature`、`--num-control-layers`、`--top-k` 或 `--chroma-bins` 必须与原实验匹配；
- 若换了输出目录，用 `--default-root-dir` 明确区分实验线。

Linux / Docker 里先确认 checkpoint 对当前容器可见：

```bash
ls -lh "/outputs/train_controlnet_dit_cqt_2000_from_base_bs4/lightning_logs/version_0/checkpoints/epoch=10-step=2000.ckpt"
```

如果宿主机上有文件，但容器内 `ls` 找不到，说明 `/outputs` 没有挂进当前容器，或这次进入的是另一个容器/会话。

### 5.3 同一 seed 下扫 `control_scale`

示例：CQT 或 chromagram 都可用，关键是保持同一个 ckpt、reference、prompt、seed，只改 `--control-scale` 和输出目录。

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\compare_controlnet_generation.py `
  --ckpt-path outputs\train_controlnet_dit_chromagram\lightning_logs\version_0\checkpoints\epoch=0-step=50.ckpt `
  --reference-audio stable_audio_control\data\song_describer\train\000000_track_id-1004034_caption_id-859.wav `
  --prompt "Lofi study beat with a clear piano melody" `
  --seconds-total 5 `
  --steps 8 `
  --seed 0 `
  --output-dir outputs\controlnet_generation_compare_chromagram_scale2 `
  --melody-feature chromagram `
  --control-scale 2.0 `
  --model-half
```

建议至少跑：

- `--control-scale 0.0`：控制路径关闭或近似关闭；
- `--control-scale 1.0`：默认控制强度；
- `--control-scale 2.0` 或 `3.0`：放大控制影响。

判断标准：

- 同 seed 下，多个目录里的 `base.wav` 应该一致；
- `control.wav` 应随 `control_scale` 变化；
- `compare.json` 里参数应能复现实验。

### 5.4 随机 10 组参考 + 10 个 seed 批量对比

脚本：

```text
stable_audio_control/scripts/compare_controlnet_generation_random.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\compare_controlnet_generation_random.py `
  --ckpt-path outputs\train_controlnet_dit\lightning_logs\version_5\checkpoints\epoch=0-step=50.ckpt `
  --reference-root stable_audio_control\data `
  --prompt "Lofi study beat with a clear piano melody" `
  --num-samples 10 `
  --random-seed 0 `
  --output-dir outputs\controlnet_generation_random `
  --model-half
```

Linux/bash 写法：

```bash
python3 stable_audio_control/scripts/compare_controlnet_generation_random.py \
  --ckpt-path outputs/train_controlnet_dit/lightning_logs/version_5/checkpoints/epoch=0-step=50.ckpt \
  --reference-root stable_audio_control/data \
  --prompt "Lofi study beat with a clear piano melody" \
  --num-samples 10 \
  --random-seed 0 \
  --output-dir outputs/controlnet_generation_random \
  --model-half
```

如果想让每条随机参考音频使用自己的 Song Describer 提示词，可以省略 `--prompt`：

```bash
python3 stable_audio_control/scripts/compare_controlnet_generation_random.py \
  --ckpt-path outputs/train_controlnet_dit/lightning_logs/version_5/checkpoints/epoch=0-step=50.ckpt \
  --reference-root stable_audio_control/data/song_describer/train \
  --num-samples 10 \
  --random-seed 0 \
  --output-dir outputs/controlnet_generation_random_reference_prompt \
  --model-half
```

如果你仍想保留一个通用 fallback prompt，但优先跟随参考音频 manifest 里的 prompt，可以加 `--use-reference-prompt`。脚本默认会查找参考音频同级 split 对应的 `manifests/<split>.json`，也可以用 `--reference-prompt-manifest <path/to/train.json>` 显式指定。

它会：

- 递归扫描 `--reference-root` 下支持的音频文件；
- 随机抽 10 个参考音频和 10 个 seed；
- 每个样本输出一条目录，里面包含 `reference.wav`、`control.wav`、`similarity.json`；
- 每条样本生成并评分后，在控制台打印 `prompt_source=<reference|argument>` 和 `similarity_score=<分数> metric=<指标名>`；
- 根目录额外写一个 `summary.json`，汇总平均分、最小分、最大分和每条样本结果。

如果你只想限制在 Song Describer 导出的训练音频里，可以把 `--reference-root` 改成 `stable_audio_control\data\song_describer\train`。

常见问题：

- `No prompt available for reference audio ...`：当前没有通用 `--prompt`，且脚本没有找到对应 manifest prompt。传入 `--prompt`，或确认存在 `song_describer/manifests/train.json`，或显式设置 `--reference-prompt-manifest`；
- `dpmpp-3m-sde requires --steps >= 2`：把 `--steps` 设为 `2` 或更高；
- 显存不足：加 `--model-half`、缩短 `--seconds-total`、降低 `--sample-size`；
- 输出无明显差异：短训 50 step 可能只证明路径有效，不代表旋律控制已学好。

## 6. 旋律相似度比较

当前更推荐直接用旋律特征比较来判断 control 音频和参考音频有多接近。这里的比较对象不是波形 RMS，而是和训练一致的 melody feature：

- `cqt`：比较 top-k pitch index overlap。
- `chromagram`：比较逐帧 chroma cosine similarity。

### 6.1 单条参考音频 vs 生成音频

脚本：

```text
stable_audio_control/scripts/compare_melody_similarity.py
```

命令：

```powershell
.\.venv\Scripts\python.exe stable_audio_control\scripts\compare_melody_similarity.py `
  --reference-audio stable_audio_control\data\song_describer\train\000000_track_id-1004034_caption_id-859.wav `
  --generated-audio outputs\controlnet_generation_compare\control.wav `
  --melody-feature cqt `
  --top-k 4 `
  --output-json outputs\controlnet_generation_compare\control_similarity.json
```

输出：

```text
outputs/controlnet_generation_compare/control_similarity.json
```

看什么：

- `similarity.metric_name`：当前用的是哪种比较方式。
- `similarity.score`：越高表示和参考旋律特征越接近。
- `feature.reference_shape` / `feature.generated_shape`：两边提取出来的旋律特征形状。

### 6.2 批量随机生成后的旋律评分

如果你已经跑了 `compare_controlnet_generation_random.py`，它会在每条样本目录里写 `similarity.json`，根目录写 `summary.json`。这比手工拷贝 wav 再一条条评分更顺手。

控制台也会直接显示每条样本的分数和最终汇总，例如：

```text
[1/50] similarity_score=0.123456 metric=cqt_topk_pitch_overlap_rate metadata=/outputs/controlnet_generation_random/00_seed-.../similarity.json
similarity_summary metric=cqt_topk_pitch_overlap_rate mean=0.234567 min=0.100000 max=0.400000
```

### 6.3 波形数值比较（可选）

`compare_controlnet_generation.py` 会在生成 `base.wav` 与 `control.wav` 后自动计算波形差异，终端打印 `audio_difference ...`，并把同样的结果写入 `compare.json` 的 `audio_difference` 字段。这个统计直接使用脚本生成后的 tensor，不需要再手动 `torchaudio.load`，因此不会触发缺少 `torchcodec` 的加载错误。

重点看这些字段：

- `max_abs_diff` / `mean_abs_diff` / `rms_diff`：越接近 0，base 与 control 越接近；
- `relative_rms_diff`：`rms_diff / base_rms`，更方便横向比较；
- `channel_correlation`：左右声道相关性，越接近 1 越相似；
- `control_scale=0` 时，理想结果是 `rms_diff` 很小、`channel_correlation` 接近 1。

下面保留旧的数值比较片段，适合比较多个不同目录里的输出，例如扫不同 `control_scale` 后的结果。

示例：比较三个 chromagram scale 输出：

```powershell
$code = @'
import soundfile as sf
import numpy as np

paths = [
    r"outputs\controlnet_generation_compare_chromagram_scale1\control.wav",
    r"outputs\controlnet_generation_compare_chromagram_scale2\control.wav",
    r"outputs\controlnet_generation_compare_chromagram_scale3\control.wav",
]
names = ["scale1", "scale2", "scale3"]
aud = [sf.read(path, always_2d=True)[0] for path in paths]

for name, audio in zip(names, aud):
    print(name, "rms=", float(np.sqrt(np.mean(audio * audio))), "peak=", float(np.max(np.abs(audio))))

for i, j in [(0, 1), (1, 2), (0, 2)]:
    diff = aud[i] - aud[j]
    print(
        names[i] + " vs " + names[j],
        "mean_abs_diff=", float(np.mean(np.abs(diff))),
        "diff_rms=", float(np.sqrt(np.mean(diff * diff))),
        "corrL=", float(np.corrcoef(aud[i][:, 0], aud[j][:, 0])[0, 1]),
        "corrR=", float(np.corrcoef(aud[i][:, 1], aud[j][:, 1])[0, 1]),
    )
'@
.\.venv\Scripts\python.exe -c $code
```

看什么：

- `rms` 不应全为 0；
- `peak` 接近 1 是正常的保存归一化现象；
- `mean_abs_diff` 和 `diff_rms` 越大，两个输出差异越明显；
- `corrL/corrR` 越低，波形越不相似；
- base 完全一致、control 随 scale 变化，说明对比实验设置是干净的。

建议结合听感判断：

- 数值差异大只说明控制路径影响了输出；
- 是否学到旋律，需要听 `control.wav` 是否跟 reference 的主旋律、音高轮廓或和声重心更接近。

## 12. 常见异常速查

| 现象 | 优先判断 | 处理方向 |
| --- | --- | --- |
| `No module named 'flash_attn'` | 可忽略警告 | 当前脚本会禁用 Flash Attention |
| Hugging Face 下载慢或超时 | 模型/数据未缓存 | 配置 `HF_TOKEN`，或先完成缓存 |
| `Cannot send a request, as the client has been closed.` | T5 tokenizer/encoder 或 StableAudio Open 文件下载/缓存失败 | 先跑“1.3 Hugging Face / T5 缓存自查”，确认 `t5-base` 和 `stabilityai/stable-audio-open-1.0` 都能加载 |
| `Checkpoint not found` 或 `FileNotFoundError ... .ckpt` | `--ckpt-path` 在当前容器/会话中不可见 | 在容器里跑 `ls -lh "<ckpt-path>"`；如果宿主机有但容器没有，检查 `/outputs` 挂载或复制 checkpoint |
| `libtorchcodec_core*.dll` 加载失败 | torchcodec/FFmpeg 兼容问题 | 走项目的 torchaudio fallback；必要时安装 full-shared FFmpeg |
| DataLoader pickling error | Windows spawn + metadata module | 确认 `custom_metadata_module` 在包路径内，训练脚本会替换成可 pickle 函数 |
| `padding_mask` 插值维度错误 | metadata mask 形状不符合上游期望 | 使用当前 `train_controlnet_dit.py` 的 `normalize_metadata_padding_masks` |
| loss 是 NaN/Inf | 数值不稳或数据异常 | 降学习率、缩短音频、先用 `32-true`、检查 wav |
| control 和 base 几乎一样 | zero-init、训练太短或 scale 太低 | 扫 `control_scale`，增加训练步数，再听感判断 |
| chromagram ckpt 加载/效果异常 | 配置和 checkpoint 不匹配 | 对比命令必须加 `--melody-feature chromagram --chroma-bins 12` |
| `No audio files were found under the reference root.` | `--reference-root` 指向的目录里没有可识别的音频 | 改成实际的音频目录，或先导出 Song Describer wav |
| `Seed range ... is too small for ... unique seeds.` | `--seed-min` / `--seed-max` 范围不够大 | 扩大 seed 范围，或把 `--num-samples` 调小 |
| 生成显存不足 | 模型大、时长长、步数多 | 加 `--model-half`、降低 `--seconds-total`、降低 `--steps` |

| `main.py` | 临时脚本 | 打印预训练模型结构 |
| `parse_model.py` | 辅助脚本 | 模型结构文本转 Markdown |

## 14. 最小验收标准

一次 ControlNet 实验至少建议留下这些证据：

1. 数据导出日志：导出了多少 wav，`dataset_config_train.json` 在哪里；
2. 训练启动日志：`melody_feature`、`control_channels`、`trainable_name_samples`；
3. 训练结束日志：达到 `max_steps`，保存 `.ckpt`；
4. 生成对比目录：`base.wav`、`control.wav`、`compare.json`；
5. 旋律相似度结果：`similarity.json` 或 `summary.json` 里的 `metric_name`、`score`、`feature.reference_shape` 和 `feature.generated_shape`；
6. 听感记录：control 是否更接近 reference 的旋律/和声/音高轮廓。

这样后续扩展到 500 step、2000 step 或更大数据集时，每条实验线都能回溯。
